## Executive Summary

The Regime Detection Engine is substantially implemented (387 of 505 requirements marked implemented), but the spec-compliance review surfaced 46 confirmed findings (13 high, 16 medium, 17 low) plus 2 uncertain, and the failures cluster precisely where the First Principle (capital protection, replayability, fast debugging) is most load-bearing: the transition-risk evidence path, the data-quality hysteresis path, and the two qualification gates. Overall posture: functionally close but NOT yet safe to promote to V2. The high-severity findings are dominated by silent correctness bugs (evidence dropped or renormalized instead of erroring) and by gates that mint "pass" verdicts without actually verifying replay, NaN-leakage, the golden suite, or the 252-session/1-year windows. Test coverage is the weakest pillar: the byte-identity claim, the golden-date regression suite, and both gates are largely unenforced.

Top risks and posture:
- Silent capital-protection bugs (HIGH): missing model-evidence is renormalized not raised (F-006); the cluster-flip term is dropped from model_instability (F-005); DQ flicker off high-risk labels is held by a flat freeze window instead of the per-label de-escalation threshold (F-038); an undeclared data_outage_specialist pre-empts crisis_specialist (F-039).
- Unenforced gates (HIGH): walk-forward gate has no replay-mismatch check (F-014), no NaN-leakage detector (F-015), trusts an external golden JSON without version match (F-016); no 252-consecutive-session shadow counter (F-018); replay mismatch never logged as a qualification-breaking incident (F-046).
- Replayability claim hollow (HIGH): test_v1_frozen_replay never runs the live engine, so byte-identity is unverified (F-004).
- Golden-date integrity (HIGH): none of the 10 §12.2 mandated dates are present (F-008); pre-2019 dates silently skipped (F-009); breadth golden test only checks enum membership (F-011).
- One non-goal violation: data_outage_specialist invents a fallback state outside spec §5.1 precedence (F-039).

=== HIGH (13) ===

F-006 (Mandatory model-evidence): resolved. §4.2/§4.0/§6 (V2L34-039, V2L56-002) now pin the rule that once transition_score is enabled, a missing HMM probability, cluster id, or change-point score raises a runtime error. `transition_risk_series.py` checks the required model-evidence stores before building the score inputs, and `transition_score.py` raises if per-session model-instability evidence is absent.

F-005 (Cluster-flip term dropped from model_instability): §6 (V2L34-038/V2L56-019) defines model_instability = max(|hmm.top_state_prob[t]-[t-5]|, change_point.score, 1.0 if cluster_id[t]!=cluster_id[t-5] else 0.0). transition_score.py:248 computes only _max_present(hmm_shift, cp_score); cluster_id params are passed but discarded, so the GMM layer contributes nothing. Fix: add the cluster-flip term before line 248 and fold it into the max.

F-038 (DQ flicker held by flat freeze window, not per-label threshold): §V2L34-021 / ADR 0010 require a transient DQ departure from a high-risk state (e.g. correlation_to_one) to be debounced by THAT label's de-escalation threshold. hysteresis.py:135 gates DQ-forced unknowns on a single global data_gap_count <= max_unknown_freeze_days. Fix: key the hold on the label being left. A premature exit from correlation_to_one/systemic_stress is a direct capital-protection regression.

F-039 (Undeclared data_outage_specialist pre-empts crisis_specialist): §5.1 enumerates 8 specialists + default_neutral headed by crisis_specialist; §2.13/§10 forbid inventing fallback states. cohort_routing.py:22-33 declares a 10th cohort at the head of COHORTS and short-circuits to it (lines 69-76) whenever any unknown-sensitive axis is unknown — so a crisis_vol session with one unknown axis routes to data_outage instead of crisis_specialist. Fix: remove the unconditional short-circuit and walk spec precedence first, only falling to a fail-closed cohort when no higher-precedence specialist matches. (Also the sole non_goal_violated finding.)

F-004 (Byte-identity unenforced): V1OUT-002 says byte-identity is enforced by test_v1_frozen_replay.py. ## 3. Untested Requirements

Testing is the weakest compliance pillar; 26 requirements are untested (V2L34-038, V2L34-039, V2L56-019, V2L56-002, V1OUT-028, V1OUT-019, V1OUT-002, V1OUT-004, V1OUT-005, V1OUT-008, V1OUT-012, V1OUT-034, V1OUT-020, V1OUT-016, V1OUT-021, V1OUT-018, WF-014, WF-015, WF-016, WF-020, WF-023, V2L56-026, V2AMB2-035, V2END-011, V2AMB2-043). The gaps cluster in three high-stakes areas.

Golden-date set (V1OUT-020/016/021/018, V2END-011): The single most damaging cluster. The committed golden_dates.yaml does not contain any of the 10 §12.2 mandated dates (F-008); pre-2019 dates are silently filtered out by the V2 fixture window (F-009); no pytest asserts all 10 dates pass with zero skips (F-010); per-axis tests skip the equality assertion whenever data_quality != 'ok' (F-012); the breadth axis only checks enum membership and never the expected value (F-011); and none of the 9 §9.4 V2-specific dates are registered (F-013). The net effect is that the regression suite the spec relies on as ground truth cannot actually catch a regime regression — an axis can flip label or degrade to insufficient_history on a golden date and every test still passes.

Byte-identity / replayability (V1OUT-002, V1OUT-028): test_v1_frozen_replay round-trips fixtures through a parallel shim model and never runs the live engine (F-004), so the central replayability guarantee is asserted but not exercised. Separately, the §2.2 window-length-independence invariant holds in practice for V1 but has no regression test (F-058).

The two gates:
- Walk-forward gate (WF-014/015/016/020/023): the gate cannot fail on a NaN leak (F-015), a replay mismatch (F-014), a stale/version-mismatched or fabricated golden result (F-016), a too-short OOS window (F-017), or any of the §8 red flags (F-054). It currently mints a pass verdict while verifying almost none of the spec's pass conditions.
- Shadow runner gate (SR-021 family, V2PRE-004): no code counts the 252 consecutive successful sessions or implements the extend/restart rules (F-018); replay mismatches are not raised as qualification-breaking incidents (F-046); the dead-man's-switch does not signal via exit code (F-047) and there is no whole-window contiguity scan (F-048). The forward-shadow prerequisite is effectively unverified end to end.

Lower-stakes untested items: the §6.1 HMM drift/transition-prob monitoring metric (V2AMB2-035/V2L56-026, F-025) and the §5.3 vol-crush exposure behavior (V2AMB2-043) have no code surface or tests.

## 4. Ambiguous Requirements

No unresolved contract ambiguities remain from the three material items in this
section.

1. V2L34-039 / V2L56-002 — resolved. Once `transition_score` is enabled,
   HMM, cluster/GMM, and change-point evidence are mandatory inputs to
   `model_instability_score`. Missing model evidence raises a runtime error;
   it is not degraded to a data-quality unknown and is not included in optional
   component renormalization. Renormalization is reserved for the optional
   components enumerated in §4.3 after the minimum coverage gate passes.
   Evidence: `docs/regime_engine_v2_spec.md` §4.0/§4.2/§6/§10,
   `src/regime_detection/transition_risk_series.py`, and
   `src/regime_detection/transition_score.py`.

2. V1GEC-018 — resolved. Escalation is a shipped configurable knob on the
   neutral axis-level hysteresis sections via `default_escalation_days`
   (default `1`) and `escalation_days_by_label`. The old flat
   `*_escalation_days` wording is reconciled to those per-axis config fields;
   default `1` preserves byte-identical immediate escalation while higher
   values delay stable-label entry and keep `active_label` risk-fast.
   Evidence: `docs/regime_engine_v1_final_spec.md` §2.10,
   `src/regime_detection/_config_layer1.py`, `src/regime_detection/hysteresis.py`,
   and `tests/test_per_label_hysteresis.py`.

3. V1L1-014 — resolved. ADX_14 uses the pandas-EWM Wilder convention:
   `ewm(alpha=1/14, adjust=False, min_periods=14)` for ATR, +DM, -DM, and DX
   smoothing. This pins warm-up byte identity to the shipped implementation
   rather than textbook simple-mean seeding. Evidence:
   `docs/regime_engine_v1_final_spec.md` §4.4 and
   `tests/test_trend_character.py::test_v1_spec_pins_adx_ewm_seeding_convention`.

The key-name inconsistency `state` vs `label` across §11/§11.1 is resolved
under F-003 by reconciling the V1 wire contract to one key across spec, shim,
and fixtures.
